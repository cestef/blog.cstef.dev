## The <GradientText>specs</GradientText>

The GameBoy is equipped with a CPU based on the `Z80`, the `LR35902`. You will find a lot of documentation on the `Z80` online, but the `LR35902` is a bit more obscure. The [GameBoy CPU Manual](http://marc.rawer.de/Gameboy/Docs/GBCPUman.pdf) is a good place to start.

To sum up the specs, the `LR35902` is an 8-bit processor with a clock speed of **4.19 MHz**. It has a **16-bit** address bus that can address up to **64KB** of memory. The CPU includes 8 general-purpose registers, a program counter, a stack pointer, and flags that are used for arithmetic operations.

One of the key features of the `LR35902` is its support for interrupts, which allows the processor to respond to external events such as button presses or timer events. The CPU also includes a range of hardware peripherals, including timers, sound generation, and LCD display controllers.

Overall, the `LR35902` is a relatively simple CPU with a limited instruction set, but it is well-suited to the requirements of the Game Boy console and is capable of running a wide range of games and applications.

## Writing the <GradientText>CPU</GradientText>

The first thing we need to do is to write the `LR35902`. The CPU is the heart of the GameBoy, and it is responsible for executing the instructions that are stored in the memory.

It is made of a few components:

-   Registers
    -   8-bit registers: `A`, `B`, `C`, `D`, `E`, `H`, `L`
    -   16-bit registers: `PC` (program counter), `SP` (stack pointer)
    -   Flags: `Z`, `N`, `H`, `C`
-   Interrupts
    -   Interrupts are handled by the CPU
    -   There are 5 interrupt types: `V-Blank`, `LCD STAT`, `Timer`, `Serial`, `Joypad`
    -   Interrupts are enabled/disabled by the `IME` flag
-   Timers

### Registers

The registers are the main components of the CPU. They are used to store data and to perform arithmetic operations. The `LR35902` CPU has 8 general-purpose **8-bit** registers, and 2 special-purpose **16-bit** registers.

Some of the registers have special purposes:

-   `A` is used for arithmetic operations
-   `F` is used to store the flags: `Z` (zero), `N` (subtract), `H` (half-carry), `C` (carry)
-   `PC` is the program counter, and it is used to store the address of the next instruction to be executed
-   `SP` is the stack pointer, and it is used to store the address of the top of the stack

The carry flag is set when there is a carry out of the most significant bit (**MSB**) of the result. For example, if you add 255 and 2 (both 8-bit numbers), the result would be 1 with the carry flag set to indicate the carry out of the **MSB**:

```text
  11111111   (255)
+ 00000010   (2)
  --------
  100000001  (1 with carry out of MSB)
      ^
      |
  carry flag set
```

The half-carry flag is set when there is a carry or borrow out of the **4th bit** (counting from the right) of the result. For example, if you add 15 and 1, the result would be 16 with the half-carry flag set to indicate the need for an adjustment between the lower and higher decimal digits:

```text
  00010101   (15)
+ 00000001   (1)
  --------
  00010110   (16)
     ^
     |
 half-carry flag set
```

The zero flag is set when the result of an operation is zero. For example, if you subtract 1 from 1, the result would be 0 with the zero flag set to indicate that the result is zero:

```text
  00000001   (1)
- 00000001   (1)
  --------
  00000000   (0)
         ^
         |
 zero flag set
```

The subtract flag is simply set when a subtraction is performed.

We will then define a `CPU` `struct{:rust}` that will hold the registers and the flags:

```rust filename="src/cpu.rs"
struct Cpu {
    // Registers
    a: u8,
    b: u8,
    c: u8,
    d: u8,
    e: u8,
    h: u8,
    l: u8,
    f: u8, // flags register
    sp: u16, // stack pointer
    pc: u16, // program counter

    // Interrupt Master Enable flag
    ime: bool,
    // Cycles left for the current instruction
    cycles: u8,
}
```

The `CPU` `struct{:rust}` will also hold the `IME` flag, which is used to enable/disable interrupts. It will also hold the `cycles` variable, which is used to keep track of the number of cycles left to execute the current instruction.

The `CPU` `struct{:rust}` will also have a `new(){:rust}` method that will initialize the registers and the flags to their default values:

```rust filename="src/cpu.rs"
impl Cpu {
    fn new() -> Self {
        Cpu {
            a: 0x01,
            b: 0x00,
            c: 0x13,
            d: 0x00,
            e: 0xd8,
            h: 0x01,
            l: 0x4d,
            f: 0xb0, // 10110000 -> Z: 1, N: 0, H: 1, C: 1
            sp: 0xfffe,
            pc: 0x0100,
            ime: false,
            cycles: 0,
        }
    }
}
```

The values of the registers are arbitrary and they were taken because they form a [valid starting point](https://gbdev.io/pandocs/Power_Up_Sequence.html#cpu-registers) for the CPU to execute instructions.

We can now write helper methods to read / write values to / from the memory.

Handling flags:


Writing to registers:

```rust filename="src/cpu.rs"
impl Cpu {
    // ...
    fn write_register(&mut self, reg: Register, data: u8) {
        match reg {
            Register::A => self.a = data,
            Register::B => self.b = data,
            Register::C => self.c = data,
            Register::D => self.d = data,
            Register::E => self.e = data,
            Register::H => self.h = data,
            Register::L => self.l = data,
            Register::F => self.f = data,
        }
    }

    fn write_word_register(&mut self, reg: Register, data: u16) {
        let (msb, lsb) = split_u16(data);
        match reg {
            Register::AF => {
                self.a = msb;
                self.f = lsb;
            }
            Register::BC => {
                self.b = msb;
                self.c = lsb;
            }
            Register::DE => {
                self.d = msb;
                self.e = lsb;
            }
            Register::HL => {
                self.h = msb;
                self.l = lsb;
            }
            Register::SP => {
                self.sp = data;
            }
            Register::PC => {
                self.pc = data;
            }
            _ => panic!("Cannot write word to 8-bit register"),
        }
    }
}
```

Reading from registers:

```rust filename="src/cpu.rs"
impl Cpu {
    // ...
    fn read_register(&self, reg: Register) -> u8 {
        match reg {
            Register::A => self.a,
            Register::B => self.b,
            Register::C => self.c,
            Register::D => self.d,
            Register::E => self.e,
            Register::H => self.h,
            Register::L => self.l,
            Register::F => self.f,
        }
    }

    fn read_word_register(&self, reg: Register) -> u16 {
        match reg {
            Register::AF => combine_u8(self.a, self.f),
            Register::BC => combine_u8(self.b, self.c),
            Register::DE => combine_u8(self.d, self.e),
            Register::HL => combine_u8(self.h, self.l),
            _ => panic!("Cannot read word from 8-bit register"),
        }
    }
}
```

`split_u16(){:rust}` and `combine_u8(){:rust}` are helper functions that will be used to split a `u16` into two `u8` and vice versa:

```rust filename="src/cpu.rs"
fn split_u16(data: u16) -> (u8, u8) {
    let msb = (data >> 8) as u8;
    let lsb = (data & 0x00ff) as u8;
    (msb, lsb)
}

fn combine_u8(msb: u8, lsb: u8) -> u16 {
    ((msb as u16) << 8) | (lsb as u16)
}
```

And finally, we can write the part that handles instructions:

```rust filename="src/cpu.rs"
impl Cpu {
    // ...
    fn fetch_byte(&mut self) -> u8 {
        let byte = self.read_byte(self.pc);
        self.pc += 1;
        byte
    }

    fn fetch_word(&mut self) -> u16 {
        let lo = self.fetch_byte() as u16;
        let hi = self.fetch_byte() as u16;
        (hi << 8) | lo
    }
}
```

The `fetch_byte(){:rust}` method will read a byte from memory and increment the `PC` register. The `fetch_word(){:rust}` method will read two consecutive bytes (**lo**w and **hi**gh), "combine" them with a left shift and a bitwise OR, and increment the `PC` register twice.

```rust filename="src/cpu.rs" {7}
            let opcode = self.fetch_byte();
            self.cycles = match opcode {
                // TODO: handle instructions
                _ => println!("Not implemented")

```

The `execute(){:rust}` method will first check if there are any cycles left to execute the current instruction. If there are no cycles left, it will fetch the next opcode and execute the corresponding instruction.

We then decrement the `cycles` variable by one each time the `execute(){:rust}` method is called.

We need to implement. **each**. **instruction**. **individually**. I will not go into details about each instruction, but I will give a brief explanation of the most important ones.

Each instruction will return the number of cycles it takes to execute. For example, the `NOP` instruction takes 4 cycles to execute:

### Notable <GradientText>instructions</GradientText>

#### `LD`

The `LD` instruction is responsible for loading data from memory to a register its name stands for "load".

For example here, the instruction `0x01` loads the immediate word (2 bytes) to the `BC` register:

```rust filename="src/cpu.rs" {8-12}
impl Cpu {
    // ...
    fn execute(&mut self) {
        if (self.cycles == 0) {
            let opcode = self.fetch_byte();
            self.cycles = match opcode {
                // ...
                0x01 => {
                    let val = self.fetch_word();
                    self.write_word_register(Register::BC, val);
                    12
                }
                _ => println!("Not implemented")
            }
        }

        self.cycles -= 1;
    }
}
```

#### `JP`

The name of this instruction stands for "jump" and is used to jump to a specific address. For example, the instruction `0xC3` jumps to the address specified by the immediate word (2 bytes).

This is the equivalent of setting the `PC` register to the result of `fetch_word(){:rust}`:

```rust filename="src/cpu.rs" {8-12}

                // ...
                0xC3 => {
                    let addr = self.fetch_word();
                    self.pc = addr;
                    16
                },

            }
```

## Working with the <GradientText>GPU</GradientText>

The GPU is responsible for rendering the pixels on the screen. It has a 160×144 pixel screen, which is 20×18 tiles. Each tile is 8×8 pixels, which means that the screen has 160×144 / 8×8 = 20×18 tiles.

Our processing unit works with two memory banks: the `VRAM` and the `OAM`. The `VRAM` is used to store the tiles that will be rendered on the screen. The `OAM` is used to store the sprites that will be rendered on the screen.

If you want to learn more about the GPU, I recommend reading [this](https://gbdev.io/pandocs/Rendering.html).
